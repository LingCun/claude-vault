## 2. 파일별 상세

### 📄 신규 #1 — `templates/sample/orderSampleCurt.html`

**기반 양식**: `templates/sample/orderSampleUI.html` + `templates/sample/orderSampleVacnt.html` 의 다이렉트 결제 패턴.

**핵심 변경/특징**:
- `payMethod = CULTUREGIFT` 고정 (hidden + 화면 표시)
- `directType = 1`, `payType = AUTH` 고정 → 컬쳐랜드 ID/PW 로그인 팝업 자동 호출
- 카드 관련 필드 (`directCardCode` / `ezpAuthCd` / `directCardQuota`) hidden 으로 두되 빈값 강제
- **컬처캐쉬 전용 필드** 추가:
  - `buyBook` (도서공연구매여부 Y/N) — dropdown
  - `bookAmount` / `shoppingAmount` — BuyBook=Y 시에만 표시 (JS 토글)
  - 합계 검증: `bookAmount + shoppingAmount = goodsAmt` (제출 직전 client-side 검사)
- 테스트 MID 기본값: `100000098m`
- 안내 문구: "다이렉트 결제 전용 — 컬쳐랜드 ID/PW 로그인 팝업 자동 호출"

**⚠ #loading 오버레이 안전장치** (2026-05-18 hotfix):
- 기존 sample 페이지 양식은 `#loading { display: block }` + `window.onpageshow` 에서 hide() 였으나, onpageshow 미실행 시 화면 전체가 흐려진 채로 멈춤.
- 해결: CSS 기본값을 `display: none` 으로 + `$(document).ready` 에서 `$('#loading').hide()` 명시 호출 (이중 안전장치).

**⚠ BUYBOOK 행 초기 숨김 보강** (2026-05-18 hotfix):
- `$("tr[data-name='BUYBOOK']").hide()` 가 어떤 이유로 미실행 되면 도서공연/일반쇼핑 금액 행이 항상 보이는 문제.
- 해결: CSS 에 `tr[data-name='BUYBOOK'] { display: none; }` 명시 → JS 호출 안 돌아도 보장됨. (JS 의 .show()/.hide() 토글은 그대로 유효)

**⚠ 다크모드 배경 강제 (white) 적용** (2026-05-18 hotfix, orderSampleCurt + cancelSampleCurt 양쪽):
- OS/브라우저 다크모드 시 body 배경이 자동으로 검정 처리되어 회색 톤 input 만 둥둥 떠 보이는 문제.
- 해결:
  - `<meta name="color-scheme" content="light">` — 브라우저에 light 모드만 지원 명시
  - CSS `html, body { background: #fff; color: #000; }` — 강제 흰 배경
- 참고: 동일 문제는 기존 `templates/sample/orderSampleUI.html` 등 다른 sample 페이지에도 잠재. 표준 톤 유지가 중요하면 동일 패턴 적용 권장 (본 step 범위 외).

**진입 URL**: `redirect.do?path=sample/orderSampleCurt.html`

### 📄 신규 #2 — `templates/sample/cancelSampleCurt.html`

**기반 양식**: `templates/test/cancelSample.html` 에서 컬처캐쉬 무관 필드 제거.

**제거된 것**:
- 부분취소 select → `partCanFlg = 0` (전체취소) hidden 으로 고정
- 환불 은행 / 환불 계좌번호 / 환불 계좌주 (가상계좌 전용 → 컬처캐쉬 무관)
- 공급가 / 부가세 / 봉사료 옵션 (현금영수증 정정용 → 컬처캐쉬는 별도 처리)

**추가된 것**:
- **망상취소 체크박스** — 체크 시 form action 을 `/receiveUrlSample.netCancel` 로 변경 (8710 전문 송신)
- 취소사유(ccMsg) 필수 검증
- 안내 문구: "컬처캐쉬는 **전체취소**만 지원합니다 (부분취소 불가)"
- 테스트 MID 기본값: `100000098m`

**진입 URL**: `redirect.do?path=sample/cancelSampleCurt.html`

## 3. 내부망 이전 절차

### Step 1. 파일 복사 (3개 파일)

| 원본 (현재 작업 PC) | 대상 (내부망 front 프로젝트) |
|---|---|
| `<작업폴더>/front/src/main/resources/templates/sample/orderSampleCurt.html` | `<내부망>/front/src/main/resources/templates/sample/orderSampleCurt.html` |
| `<작업폴더>/front/src/main/resources/templates/sample/cancelSampleCurt.html` | `<내부망>/front/src/main/resources/templates/sample/cancelSampleCurt.html` |
| `<작업폴더>/front/src/main/resources/static/sample.html` | `<내부망>/front/src/main/resources/static/sample.html` |

> 첫 두 개는 **신규 파일** (덮어쓰기 위험 없음).
> 세 번째는 **기존 파일 수정** — 내부망의 `sample.html` 이 본 문서 기준과 다르면 메뉴 행 추가 부분만 발췌해서 merge 권장.

```bash
cd <내부망>/front
./gradlew clean build
# 또는 IDE 에서 hot reload (thymeleaf 는 cache 끈 환경이면 reload 만으로도 반영)
```

### Step 3. 검증

| 단계 | 확인 |
|---|---|
| 1 | https://pay-dev.fingerservice.co.kr/sample.html 접속 |
| 2 | 맨 아래에 「문화상품권」 행 보이는지 확인 |

| 4 | 「취소」 클릭 → `…/redirect.do?path=sample/cancelSampleCurt.html` 로 이동, 취소 폼 로드 |

| 6 | 취소 폼에서 망상취소 체크박스 노출 확인 |

> 실제 결제하기/취소하기 클릭 시점은 백엔드 (`CultureCashService` mock 모드) 까지 만들어진 후에 검증.
