> **선행 조건**: [[2026-05-18_컬처캐쉬_DB셋업_v02]] 적용 완료 (Phase 1) — `TBSI_COM_CD` 에 SPM_CD=07 등록되어 있어야 함.

## 2. 파일별 상세

### ✏ 수정 #1 — `Constants.java`

**위치**: `SPM_CD_05_NONE` 직후

**diff**:
```java
 	String	SPM_CD_05_NONE						= "05";											// 키인(비인증)
+	String	SPM_CD_07_CULTURE						= "07";											// 컬처캐쉬(컬쳐랜드 IDPW)

 	String	SPM_CD_AUTH_01						= "AUTH";										// 인증
 	...
 	String	SPM_CD_NONE_05						= "NONE";										// 키인(비인증)
+	String	SPM_CD_CULT_07						= "CULTURE";									// 컬처캐쉬(컬쳐랜드 IDPW)
```

**근거**: [[2026-05-18_컬처캐쉬_연동_분석보고서_v01.0]] §"SPM_CD_CULTURE, DESC1='CULTURE'" 규약. 기존 SPM_CD 06 은 미사용 자리지만 분석 보고서가 07 을 점유하도록 지정 → 변경 없이 따라감.

---

### ✏ 수정 #2 — `CultureCashService.java`

**위치**: `doStart(Map<String, Object> inParam)` 메서드 내 2개소.

**diff**:
```java
     public Map<String, Object> doStart(Map<String, Object> inParam) throws Exception {
-        // SubPM 코드: IDPW 인증 결제이므로 인증(01) 으로 매핑
-        inParam.put("spmCd", Constants.SPM_CD_01_AUTH);
+        // SubPM 코드: 컬처캐쉬(컬쳐랜드 IDPW) 전용 코드 07(CULTURE)
+        inParam.put("spmCd", Constants.SPM_CD_07_CULTURE);
         ...
         Map<String, Object> ptnQuery = new HashMap<>();
         ptnQuery.put("mid",         inParam.get("mid"));
         ptnQuery.put("pmCd",        Constants.PM_CD_32_CULTURE);
-        ptnQuery.put("spmCd",       Constants.SPM_CD_01_AUTH);
+        ptnQuery.put("spmCd",       Constants.SPM_CD_07_CULTURE);
         ptnQuery.put("serviceCode", PTN_CD_CULT);
```

**효과**:
- BLD 로 송신되는 spmCd 가 컬처캐쉬 식별 가능한 `07` 로 바뀜.
- `getShopPtnKey` 의 `TBSI_PTN_CPID` 조회 키도 `(MID, PM_CD=32, SPM_CD=07, SERVICE_CODE='CULTURE')` 4-튜플 매칭으로 정확해짐 (DB v03 SQL 의 가맹점 등록 키와 일치).
