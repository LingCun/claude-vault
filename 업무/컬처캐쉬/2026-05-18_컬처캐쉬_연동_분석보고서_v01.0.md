## 1. 분석 개요

### 1.1 목적

기존 핑거페이 인증결제(신용카드/가상계좌) 인프라에 **컬처캐쉬(컬쳐랜드 IDPW)** 결제수단을 추가할 때 필요한 DB 테이블 영향도 및 신규 테이블 명세를 정의한다.

### 1.2 범위

- FRONT: 가맹점 → 핑거페이 결제 요청 / 컬쳐랜드 로그인 팝업 / 인증 결과 적재 / 가맹점 응답
- BLD-v3: 컬쳐랜드 8110/8120/8210/8220/8410/8420/8710/8720 전문 송수신 및 거래 적재
- 본 보고서는 **DB 테이블 영향**에 집중하며, 매퍼/서비스 클래스 신설은 별도 구현 plan에서 다룬다.

### 1.3 참고 자료

- `핑거페이_가맹점연동가이드_인증결제_v0.9.4.docx`
- `FW_ Fwd_ 컬쳐캐시 연동 가이드/컬쳐캐쉬연동 전문 (IDPW 방식)/IDPW방식_결제전문.doc`
- `웹_컬쳐랜드로그인전문.docx`, `모바일웹_컬쳐랜드로그인전문.docx`
- `front/src/main/java/solpay/wiezon/com/payMethod/service/CultureCashService.java` (현재 구현 step1)
- `bld-v3/src/main/resources/mapper/*.xml` (card.xml, common.xml 등)

## 2. 컬처캐쉬 처리 흐름

### 2.1 End-to-End 시퀀스

```
[가맹점]                  [FRONT]                   [컬쳐랜드]              [BLD-v3]              [한국문화진흥]
   │  ① POST /payment/v1/view/request               │                       │                      │
   │ ─────────────────────► │                       │                       │                      │
   │                        │ ② TBTR_REQ INSERT     │                       │                      │
   │                        │ ③ POST 컬쳐랜드        │                       │                      │
   │                        │   weblogin.do          │                       │                      │
   │                        │ ─────────────────────► │                       │                      │
   │                        │                        │ (사용자 ID/PW + 잔액조회)                    │
   │                        │ ◄───────────────────── │                       │                      │
   │                        │ ④ ReturnUrl 콜백       │                       │                      │
   │                        │   (RetVal/HashNo/      │                       │                      │
   │                        │    McashAmount...)     │                       │                      │
   │                        │ ⑤ TBAT_CURT INSERT     │                       │                      │
   │                        │ ⑥ TBTR_REQ UPDATE      │                       │                      │
   │ ◄ ⑦ returnUrl(인증결과)│                        │                       │                      │
   │  ⑧ POST approvalUrl    │                        │                       │                      │
   │ ─────────────────────► │                        │                       │                      │
   │                        │ ⑨ approval → BLD       │                       │                      │
   │                        │ ────────────────────────────────────────────► │                      │
   │                        │                        │                       │ ⑩ TBTR_MSTR INSERT   │
   │                        │                        │                       │ ⑪ TBTR_CURT INSERT   │
   │                        │                        │                       │ ⑫ TBTR_HIST INSERT   │
   │                        │                        │                       │ ⑬ 8110 송신 ──────► │
   │                        │                        │                       │                      │
   │                        │                        │                       │ ◄────── 8120 응답   │
   │                        │                        │                       │ ⑭ TBTR_MSTR UPDATE   │
   │                        │                        │                       │ ⑮ TBTR_CURT UPDATE   │
   │                        │                        │                       │   (실패 시 TBUS_CURT)│
   │                        │ ◄──────────────────────────────────────────── │                      │
   │ ◄ ⑯ 승인 결과          │                        │                       │                      │
```

### 2.2 단계별 DB 액션 요약

| # | 단계 | 위치 | 테이블 | 액션 |
|---|---|---|---|---|
| ② | 결제 요청 적재 | FRONT | `TBTR_REQ` | INSERT |
| ⑤ | 컬쳐랜드 인증 결과 적재 | FRONT | `TBAT_CURT` (신규) | INSERT |
| ⑥ | 인증 결과 반영 | FRONT | `TBTR_REQ` | UPDATE (`AUTH_TYPE_CD` 등) |
| ⑩ | 거래 마스터 생성 | BLD | `TBTR_MSTR` | INSERT |
| ⑩ | 결제 요청 상태 | BLD | `TBTR_PAY_REQ` | UPDATE |
| ⑩ | 상품 판매량 누적 | BLD | `TBMC_GOODS` | UPDATE |
| ⑪ | 컬처캐쉬 거래 원장 | BLD | `TBTR_CURT` (신규) | INSERT |
| ⑫ | 거래 히스토리 | BLD | `TBTR_HIST` | INSERT |
| ⑫ | 분산 락 | BLD | `TBAD_CLST_VCS` | INSERT |
| ⑭ | 거래 마스터 상태 갱신 | BLD | `TBTR_MSTR` | UPDATE |
| ⑮ | 거래 원장 갱신 / 실패 적재 | BLD | `TBTR_CURT` / `TBUS_CURT` (신규) | U / I |
| 후속 | 정산 적재 | BLD | `TBCC_DPST_REQ` | INSERT |
| 후속 | 현금영수증 (BuyBook=Y) | BLD | `TBTR_NTS_CRCT_REQ` | INSERT |
| 취소 | 취소 이력 (전체취소/망취소만) | BLD | `TBCC_REVOKE_HIST` | INSERT |
| 취소 | 가맹점 통보 | FRONT | `TBTR_NOTI_SND` | UPDATE |

> **취소 정책:** 컬처캐쉬는 **전체취소(8210) 및 망상취소(8710) 만 지원**한다. 부분취소(8410)는 제공하지 않는다.

## 3. 테이블 영향 분석

### 3.1 공용 사용 가능 (변경 없이 재사용)

| 테이블 | 위치 | 분기 키 | 비고 |
|---|---|---|---|
| `TBTR_REQ` | FRONT | `PM_CD=32`, `SPM_CD=07` | NONCE 기반 요청 적재 — 카드 컬럼(`CARD_NO`, `QUOTA_MON` 등)은 NULL 처리 |
| `TBTR_MSTR` | BLD | `PM_CD=32` | 거래 마스터, `PTN_CD='CULT'` |
| `TBTR_PAY_REQ` | BLD | - | 결제 요청 상태 |
| `TBMC_GOODS` | BLD | - | 상품 판매량 누적 |
| `TBTR_HIST` | BLD | - | 거래 히스토리 |
| `TBAD_CLST_VCS` | BLD | - | 중복결제 방지 락 |
| `TBTR_NOTI_SND` | FRONT | - | 가맹점 결과 통보 |
| `TBUS_MSTR` | BLD | - | 결제 실패 공용 원장 |
| `TBCC_DPST_REQ` | BLD | - | 정산 (정산주기 설정 시) |
| `TBCC_REVOKE_HIST` | BLD | - | 전체취소(8210) / 망취소(8710) 이력 |
| `TBTR_NTS_CRCT_REQ` | BLD | - | 컬쳐랜드 `BuyBook=Y` 시 그대로 사용 |
| `TBTR_CRCT_TRX` / `TBUS_CRCT` | BLD | - | 현금영수증 정정 |

### 3.2 미사용 (스킵)

| 테이블 | 사유 |
|---|---|
| `TBAT_AUTH` | ISP/3D 인증 전용 컬럼 (`XID`, `CAVV`, `ECI`). 컬처캐쉬는 컬쳐랜드 IDPW 팝업이라 미사용 |
| `TBTR_CARD` | 카드사 승인번호/할부/매입 정보 — 컬처캐쉬와 무관 |
| `TBUS_CARD` | 카드 실패 컬럼(`CARD_RSLT_CD` 등) |
| `TBBL_MSTR` | 카드 빌링/자동결제 — 컬처캐쉬 빌링 미지원 |
| `TBCI_EVNT_CARD` | 무이자 이벤트 — 컬처캐쉬 무관 |
| `TBTR_RFND` | 가상계좌 환불 전용 |

### 3.3 신규 필요 (3종)

| 테이블 | 위치 | 대응 (카드) | 역할 |
|---|---|---|---|
| **`TBAT_CURT`** | FRONT | `TBAT_AUTH` | 컬쳐랜드 로그인 팝업 인증 결과 (HashNo, McashAmount 등) 적재 |
| **`TBTR_CURT`** | BLD | `TBTR_CARD` | 컬쳐랜드 8120 승인 응답(거래번호, 승인금액, 도서공연 정보 등) 적재 |
| **`TBUS_CURT`** | BLD | `TBUS_CARD` | 컬쳐랜드 8120 / 8220 / 8720 실패 응답(RetVal, ErrMsg) 적재 |

> **부분취소 미지원.** 망상취소(8710) 응답은 `TBCC_REVOKE_HIST` 에 `NET_CAN_FLG='1'` 으로 적재하며, 별도 신규 테이블은 두지 않는다.

## 4. CURT 신규 테이블 명세서

### 4.1 명명 규칙

핑거페이 기존 컨벤션을 따른다.

| 영역 | 컨벤션 |
|---|---|
| 테이블명 | `TB` + 영역(`AT`/`TR`/`US`/`CC`/`BL`/`MC`) + `_` + 결제수단 단축명 |
| 결제수단 단축명 | 카드 = `CARD`, 가상계좌 = `VACNT`, **컬처캐쉬 = `CURT`** |
| 컬럼명 | UPPER_SNAKE_CASE |
| 거래 PK | `TID` (NONCE→TID 매핑) |
| 등록 일시 | `REG_DNT` (단일 컬럼) |
| 작업자 | `WORKER` |
| 암호화 플래그 | `CRYPT_FLG` |
| 거래 상태 | `TRX_ST_CD` (0:승인 / 1:전취소 / 2:후취소) |
| 결과 코드 | `RSLT_CD` / `RSLT_MSG` (공용) — 결제수단 자체 코드는 prefix(`CARD_`, `CURT_`) |
| 암호화 컬럼 | 평문 컬럼명 + `_ENC` suffix |

---

### 4.2 TBAT_CURT (FRONT - 컬쳐랜드 로그인 인증 결과)

**목적:** 컬쳐랜드 로그인 팝업의 호출 정보(LoginHashCode 생성 원본)와 콜백 응답(HashNo/McashAmount 등) 을 NONCE/TID 기준으로 적재. `TBAT_AUTH` 와 동일한 위치/시점에서 사용.

**DDL (Oracle 기준):**

```sql
CREATE TABLE TBAT_CURT (
    TID             VARCHAR2(30)    NOT NULL,                -- 핑거페이 거래 ID (PK)
    MID             VARCHAR2(10)    NOT NULL,                -- 상점 ID
    NONCE           VARCHAR2(30)    NOT NULL,                -- 결제 요청 NONCE
    CP_CD           VARCHAR2(10),                            -- 결제사 코드 (= 'CULT')
    PTN_CD          VARCHAR2(10),                            -- 파트너 코드 (= 'CULT')
    VAN_CD          VARCHAR2(10),                            -- VAN 코드 (NULLABLE)

    -- 컬쳐랜드 로그인 호출 파라미터
    MBR_CD          VARCHAR2(12)    NOT NULL,                -- 컬쳐랜드 MemberCode (가맹점 코드)
    LOGIN_DT        VARCHAR2(8),                             -- LoginDate (yyyyMMdd)
    LOGIN_TM        VARCHAR2(6),                             -- LoginTime (HHmmss)
    LOGIN_HASH_CD   VARCHAR2(32),                            -- LoginHashCode (md5)
    RETURN_URL      VARCHAR2(500),                           -- ReturnUrl
    USER_IP         VARCHAR2(40),                            -- UserIp (IPv4/IPv6)
    REQ_AMT         NUMBER(15,0),                            -- Amount (결제 요청 금액)

    -- 컬쳐랜드 로그인 콜백 응답
    AUTH_RSLT_CD    VARCHAR2(4),                             -- RetVal (0000 정상 / 9999 사용자취소)
    CURT_USER_ID    VARCHAR2(12),                            -- UserID (컬쳐랜드 ID)
    CURT_CUST_ID    VARCHAR2(8),                             -- CustID (컬쳐랜드 고객키)
    HASH_NO         VARCHAR2(25),                            -- HashNo (결제인증키, 1회성)
    MCASH_AMT       NUMBER(15,0),                            -- McashAmount (인증 시점 잔액)
    USE_AMT         NUMBER(15,0),                            -- RequestAmount (사용자가 입력한 사용 금액)

    -- 핑거페이 공통
    CRYPT_FLG       VARCHAR2(1)     DEFAULT 'N',
    WORKER          VARCHAR2(20),
    REG_DNT         DATE            DEFAULT SYSDATE,

    CONSTRAINT PK_TBAT_CURT PRIMARY KEY (TID)
);

CREATE INDEX IX_TBAT_CURT_01 ON TBAT_CURT (NONCE);
CREATE INDEX IX_TBAT_CURT_02 ON TBAT_CURT (MID, REG_DNT);
```

**컬럼 명세:**

| 컬럼 | 타입 | NULL | 설명 | 비고 |
|---|---|---|---|---|
| `TID` | VARCHAR2(30) | N | 거래 ID | PK |
| `MID` | VARCHAR2(10) | N | 상점 ID | TBAT_AUTH 동일 |
| `NONCE` | VARCHAR2(30) | N | 요청 NONCE | TBTR_REQ 와 매핑 |
| `CP_CD` / `PTN_CD` | VARCHAR2(10) | Y | 파트너 코드 | `'CULT'` 고정 |
| `VAN_CD` | VARCHAR2(10) | Y | VAN 코드 | 컬처캐쉬는 미사용 가능 |
| `MBR_CD` | VARCHAR2(12) | N | 컬쳐랜드 MemberCode | 가맹점별 컬쳐랜드 가맹점 코드 |
| `LOGIN_DT` | VARCHAR2(8) | Y | 로그인 호출 날짜 | yyyyMMdd |
| `LOGIN_TM` | VARCHAR2(6) | Y | 로그인 호출 시각 | HHmmss |
| `LOGIN_HASH_CD` | VARCHAR2(32) | Y | LoginHashCode | md5 결과 |
| `RETURN_URL` | VARCHAR2(500) | Y | 가맹점 리턴 URL | |
| `USER_IP` | VARCHAR2(40) | Y | 사용자 IP | LoginHashCode 생성 원본 |
| `REQ_AMT` | NUMBER(15) | Y | 요청 금액 | LoginHashCode 생성 원본 |
| `AUTH_RSLT_CD` | VARCHAR2(4) | Y | 컬쳐랜드 RetVal | `0000` / `9999` |
| `CURT_USER_ID` | VARCHAR2(12) | Y | 컬쳐랜드 ID | |
| `CURT_CUST_ID` | VARCHAR2(8) | Y | 컬쳐랜드 고객키 | 취소 시 사용 |
| `HASH_NO` | VARCHAR2(25) | Y | 결제인증키 | **승인 8110 송신 시 필수** |
| `MCASH_AMT` | NUMBER(15) | Y | 컬처캐쉬 잔액 | 인증 시점 |
| `USE_AMT` | NUMBER(15) | Y | 컬처캐쉬 사용 금액 | 8110 의 RequestAmount |
| `CRYPT_FLG` | VARCHAR2(1) | Y | 암호화 플래그 | 'Y' / 'N' |
| `WORKER` | VARCHAR2(20) | Y | 작업자 | |
| `REG_DNT` | DATE | Y | 등록 일시 | DEFAULT SYSDATE |

### 4.3 TBTR_CURT (BLD - 컬처캐쉬 거래 원장)

**목적:** 한국문화진흥에 송신한 8110 (승인) / 8410 (부분취소후재승인) 응답(8120 / 8420)을 거래 단위로 적재. `TBTR_CARD` 와 동일한 위치/시점에서 사용.

**DDL (Oracle 기준):**

```sql
CREATE TABLE TBTR_CURT (
    TID             VARCHAR2(30)    NOT NULL,                -- 핑거페이 거래 ID (PK)
    TR_DT           VARCHAR2(8),                             -- 거래일 (yyyyMMdd)
    TRX_ST_CD       VARCHAR2(1),                             -- 거래 상태 (0:승인 1:전취소 2:후취소)

    -- 핑거페이 공통 / VAN
    PTN_CD          VARCHAR2(10),                            -- 파트너 코드 (= 'CULT')
    VAN_CD          VARCHAR2(10),                            -- VAN 코드
    EDI_NO          VARCHAR2(20),                            -- 전문번호 (8120 응답)
    APP_NO          VARCHAR2(20),                            -- 승인번호 (= ControlCode 매핑)
    APP_AMT         NUMBER(15,0),                            -- 승인금액 (AuthAmount)
    AUTH_TYPE_CD    VARCHAR2(2),                             -- 인증 유형 (= 'IP' IDPW)

    -- 컬쳐랜드 8110/8120 매핑
    MBR_CD          VARCHAR2(12),                            -- MemberCode
    MBR_CTRL_CD     VARCHAR2(25),                            -- MemberControlCode (업체 거래번호)
    CTRL_CD         VARCHAR2(20),                            -- ControlCode (컬쳐랜드 거래번호)
    CURT_USER_ID    VARCHAR2(12),                            -- UserID
    CURT_CUST_ID    VARCHAR2(8),                             -- CustID
    HASH_NO         VARCHAR2(25),                            -- HashNo (사용된 인증키)
    LEVY_DT         VARCHAR2(8),                             -- LevyDate (yyyyMMdd)
    LEVY_TM         VARCHAR2(6),                             -- LevyTime (HHmmss)
    LEVY_AMT        NUMBER(9,0),                             -- LevyAmount (사용량)
    LEVY_UNIT       VARCHAR2(4),                             -- Unit (건/분/시간)
    CTNT_CD         VARCHAR2(15),                            -- ContentsCode (상품코드)
    CTNT_NM         VARCHAR2(30),                            -- ContentsName (상품명)
    GENRE_CD        VARCHAR2(2),                             -- Genre
    GENRE_DTL_CD    VARCHAR2(3),                             -- GenreDetail
    PAY_TYPE        VARCHAR2(2),                             -- PayType (00:제휴결제)

    -- 도서공연 (현금영수증)
    BUY_BOOK_FLG    VARCHAR2(1)     DEFAULT 'N',             -- BuyBook (Y/N)
    BOOK_AMT        NUMBER(9,0)     DEFAULT 0,               -- BookAmount
    SHOP_AMT        NUMBER(9,0)     DEFAULT 0,               -- ShoppingAmount

    -- 결제 후 잔액 (참고)
    MCASH_AMT       NUMBER(15,0),                            -- 결제 후 컬처캐쉬 잔액

    -- 핑거페이 공통
    CRYPT_FLG       VARCHAR2(1)     DEFAULT 'N',
    WORKER          VARCHAR2(20),
    REG_DNT         DATE            DEFAULT SYSDATE,

    CONSTRAINT PK_TBTR_CURT PRIMARY KEY (TID)
);

CREATE INDEX IX_TBTR_CURT_01 ON TBTR_CURT (CTRL_CD);
CREATE INDEX IX_TBTR_CURT_02 ON TBTR_CURT (MBR_CTRL_CD);
CREATE INDEX IX_TBTR_CURT_03 ON TBTR_CURT (TR_DT, MBR_CD);
```

### 4.4 TBUS_CURT (BLD - 컬처캐쉬 실패 원장)

**목적:** 컬쳐랜드 8120 (승인 실패) / 8220 (취소 실패) / 8720 (망취소 실패) 응답을 적재. `TBUS_CARD` 와 동일한 위치/시점에서 사용.

**DDL (Oracle 기준):**

```sql
CREATE TABLE TBUS_CURT (
    TID             VARCHAR2(30)    NOT NULL,                -- 핑거페이 거래 ID
    LOG_ID          NUMBER(15)      NOT NULL,                -- 실패 로그 일련번호 (시퀀스)

    -- 핑거페이 공통 / VAN
    PTN_CD          VARCHAR2(10),                            -- 파트너 코드 (= 'CULT')
    VAN_CD          VARCHAR2(10),                            -- VAN 코드
    EDI_NO          VARCHAR2(20),                            -- 전문번호
    AUTH_TYPE_CD    VARCHAR2(2),                             -- 인증 유형 (= 'IP')

    -- 컬쳐랜드 송신 정보 (실패한 8110/8210/8710 송신값)
    MBR_CD          VARCHAR2(12),                            -- MemberCode
    MBR_CTRL_CD     VARCHAR2(25),                            -- MemberControlCode
    LEVY_AMT        NUMBER(9,0),                             -- 시도한 RequestAmount/CancelAmount
    BUY_BOOK_FLG    VARCHAR2(1),                             -- BuyBook
    BOOK_AMT        NUMBER(9,0),                             -- BookAmount
    SHOP_AMT        NUMBER(9,0),                             -- ShoppingAmount

    -- 컬쳐랜드 응답 실패 정보
    CURT_RSLT_CD    VARCHAR2(4),                             -- RetVal (XXXX 실패 코드)
    CURT_RSLT_MSG   VARCHAR2(100),                           -- ErrMsg (실패 사유)

    -- 핑거페이 공통
    CRYPT_FLG       VARCHAR2(1)     DEFAULT 'N',
    WORKER          VARCHAR2(20),
    REG_DNT         DATE            DEFAULT SYSDATE,

    CONSTRAINT PK_TBUS_CURT PRIMARY KEY (TID, LOG_ID)
);

CREATE INDEX IX_TBUS_CURT_01 ON TBUS_CURT (REG_DNT);
CREATE INDEX IX_TBUS_CURT_02 ON TBUS_CURT (CURT_RSLT_CD);
```

| `MBR_CTRL_CD` | VARCHAR2(25) | Y | MemberControlCode | |

### 5.6 8710 / 8720 망상취소 (전체취소만)

송수신 항목은 **8210 / 8220 와 동일 매핑** 사용 (단, `HeadNo=8710`/`8720`). 응답 적재 시 `TBCC_REVOKE_HIST.NET_CAN_FLG='1'` 처리.

| **BLD 코드** | `card.xml` 와 동일 위치에 `curt.xml` 신설. `CardService` 옆 `CultureCashService` (BLD) 신설. 8110/8210/8710 socket 송수신은 기존 socket 인프라 재사용 (8410 부분취소 미사용). |
| **공용 적재 로직** | `common.xml` 의 `saveMstr` / `saveHist` / `saveTbusMstr` / `insertDepositReq` 는 PM_CD 분기만 추가하면 그대로 사용. |
| **취소 정책** | 전체취소 + 망상취소만 지원. 가맹점 취소요청 시 `partCanFlg=1` 들어오면 거절(`F211`/`2028`). |
| **현금영수증** | `BuyBook=Y` 케이스에 한해 기존 `TBTR_NTS_CRCT_REQ` 로직 재사용. |

### 7.2 리스크 / 주의 사항

1. **HASH_NO 1회성** — 컬쳐랜드 HashNo 는 인증 완료 후 1회만 유효. 인증 → 승인 사이 시간/오류로 만료될 수 있음. `TBAT_CURT.REG_DNT` 기준 timeout (예: 10분) 검증 필요.
2. **부분취소 미지원** — 컬쳐랜드 사양상 8410(부분취소) 미사용. 가맹점 취소요청 시 `partCanFlg=1` 들어오면 명시적으로 거절(`F211`/`2028`). 정책 확정 후 가맹점 가이드 v01.0 의 §4.4 (부분취소) 항목도 동일하게 삭제 필요.
3. **SEED CBC 암호화 구간** — 8110/8210/8710 의 음영부분 외 SEED CBC 암호화 처리 필요 (인코딩 EUC-KR). BLD 송수신 인프라에 SEED 모듈 적용 확인 필요.
4. **EUC-KR 인코딩** — 컬쳐랜드 socket 통신은 EUC-KR. ContentsName(상품명) 한글 byte 길이 30 초과 시 절단 정책 정의 필요.
5. **잔액 부족 처리** — 사용자가 컬쳐랜드 팝업에서 사용 금액(RequestAmount) 을 결정하므로, GOODS_AMT 와 RequestAmount 가 다를 가능성. 본 분석은 **GOODS_AMT = RequestAmount** 로 가정. 차액(부분 결제) 정책 결정 필요.
6. **컬쳐랜드 자체 응답 코드** — 핑거페이 결과 코드와 별개로 컬쳐랜드 자체 코드(`[최신]쇼핑몰에러코드정의.pptx`) 를 `TBUS_CURT.CURT_RSLT_CD` 에 그대로 적재. 핑거페이 결과 코드 매핑 테이블(`TBCM_CD`) 에 컬쳐랜드 코드 추가 등재 검토 필요.
7. **MBR_CD (MemberCode) 등록 경로** — `TBPT_PTN_KEY` (또는 동등 가맹점-파트너 키 테이블) 에 `PTN_CD='CULT'` 로 등록되어야 함. 현재 코드(`CultureCashService.java`)는 `subPgMapper.getShopPtnKey` 로 조회 중이므로 운영 등록 절차 별도 정의 필요.
8. **모바일/PC 팝업 분기** — 현재 FRONT 코드는 `mode` 환경변수로 운영/개발 URL 분기 + `userAgent` 로 PC/Mobile 분기. 이는 팝업 사이즈/URL 차이만 있으므로 DB 영향은 없음.

### 9.6 확인 쿼리 (등록 검증)

```sql
-- 가맹점별 컬처캐쉬 등록 상태 종합 조회
SELECT  s.MID,
        s.PM_CD,
        s.SPM_CD,
        l.PTN_CD,
        l.PTN_CPID                  AS MBR_CD,
        c.DESC1                     AS PM_NM,
        p."DESC"                    AS PTN_NM,
        s.USE_FLG                   AS SVC_USE,
        l.FR_DT, l.TO_DT
  FROM  TBSI_MBS_SVC      s
  LEFT  JOIN TBSI_MBS_PTN_LNK l ON s.MID = l.MID  AND s.PM_CD = l.PM_CD AND s.SPM_CD = l.SPM_CD
  LEFT  JOIN TBSI_PTN_CPID    p ON l.PTN_CD = p.PTN_CD AND l.PTN_CPID = p.PTN_CPID
  LEFT  JOIN TBAD_CODE        c ON c.COL_NM = 'PM_CD' AND c.CODE1 = s.PM_CD
 WHERE  s.MID    = 'demotest0m'
   AND  s.PM_CD  = '32'
   AND  s.USE_FLG = '1';
-- 기대: 1 row, SPM_CD='07', MBR_CD='TESTCURT0001', PM_NM='컬처캐쉬', PTN_NM='컬쳐랜드 테스트 가맹점'
```
