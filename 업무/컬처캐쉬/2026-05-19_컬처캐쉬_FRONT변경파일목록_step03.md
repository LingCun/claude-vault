## 2. 파일별 상세

### 🆕 신규 #1 — `mapper/CultureMapper.java`

**네임스페이스**: `solpay.wiezon.com.mapper.CultureMapper`

**메서드 4종**:
```java
void insertTbatCulture(Map<?, ?> map);          // 로그인 호출 시점 INSERT
int  updateTbatCulture(Map<?, ?> map);          // 콜백 응답 UPDATE
int  checkTbatCulture(Map<?, ?> map);           // TID 기준 중복체크
Map<String, Object> selectTbatCulture(Map<?, ?> map); // 승인전문 송신용 SELECT
```

---

### 🆕 신규 #2 — `mapper/CultureMapper.xml`

| statement | 시점 | 동작 | 비고 |
|---|---|---|---|
| `insertTbatCulture` | `CultureCashService.doStart()` 끝에서 호출 예정 | TID/MID/NONCE + 로그인 호출 파라미터 (MBR_CD, LOGIN_DT/TM, LOGIN_HASH_CD, RETURN_URL, USER_IP, REQ_AMT) 적재 | WORKER='FRONT', CRYPT_FLG='N', REG_DNT=NOW() |
| `updateTbatCulture` | `/cultureReturn.do` 콜백 컨트롤러 (Step 04) | AUTH_RSLT_CD + (성공 시) CULTURE_USER_ID/CULTURE_CUST_ID/HASH_NO/MCASH_AMT/USE_AMT | TID 또는 NONCE 기준 (둘 다 binding 가능) |
| `checkTbatCulture` | INSERT 직전 안전장치 | TID 중복 row 카운트 | `AuthMapper.checkTbatAuth` 와 동일 사용처 |
| `selectTbatCulture` | 승인(8110) 전 BLD 송신 전 | TBAT_CULTURE 의 모든 컬럼 SELECT → BLD 로 전달 | TID/NONCE 둘 다 binding 가능 |

**컬럼 매핑 출처**: [[2026-05-18_컬처캐쉬_연동_분석보고서_v01.0]] §4.2 (TBAT_CULTURE DDL).
