## 2. 파일별 상세

### ✏ 수정 #1 — `CultureCashService.java`

**핵심 변경 3가지**:

1. **`CultureMapper` 의존성 추가**
   ```java
   import solpay.wiezon.com.mapper.CultureMapper;
   ...
   @Autowired
   private CultureMapper cultureMapper;
   ```

2. **`returnUrl` 변경** — 컬쳐랜드 콜백은 NONCE/TID 를 자체 전달하지 않으므로 쿼리스트링에 NONCE 를 박아 식별:
   ```java
   String returnUrl = WiezonUtil.null2str(domain, "") + "/cultureReturn.do?nonce=" + nonce;
   ```

3. **TBAT_CULTURE INSERT** — 로그인 호출 시점에 인증요청 row 적재 (콜백 UPDATE 의 전제):
   ```java
   if (StringUtils.isNotEmpty(tid)) {
       Map<String, Object> insertParam = new HashMap<>();
       insertParam.put("tid",          tid);
       insertParam.put("mid",          inParam.get("mid"));
       insertParam.put("nonce",        nonce);
       insertParam.put("cpCd",         PTN_CD_CULT);   // 'CULT'
       insertParam.put("ptnCd",        PTN_CD_CULT);   // 'CULT'
       insertParam.put("vanCd",        null);
       insertParam.put("mbrCd",        memberCode);
       insertParam.put("loginDt",      loginDate);
       insertParam.put("loginTm",      loginTime);
       insertParam.put("loginHashCd",  loginHashCode);
       insertParam.put("returnUrl",    returnUrl);
       insertParam.put("userIp",       userIp);
       insertParam.put("reqAmt",       amount);
       cultureMapper.insertTbatCulture(insertParam);
   }
   ```

**주의**: `tid` 가 비어 있으면 INSERT 를 skip (PK NULL 회피).

---

### 🆕 신규 #2 — `controller/CultureReturnController.java`

**엔드포인트**: `@RequestMapping(value = "/cultureReturn.do", method = {POST, GET})`

**처리 흐름**:
1. `request` 에서 form/query 파라미터 → Map 변환
2. 쿼리스트링의 `nonce` 로 `CultureMapper.selectTbatCulture` 조회 → MBR_CD 회수
3. **MOCK 분기**: `MBR_CD == 'MOCKCULTURE1'` 이면 컬쳐랜드 응답을 무시하고 고정 stub 값 적재
   - `RetVal = '0000'`
   - `UserID = 'MOCKUSER'`, `CustID = 'MOCKCUS1'`
   - `HashNo = "MK" + NONCE` (25자 패딩)
   - `McashAmount = 1,000,000`
   - `RequestAmount = REQ_AMT` (TBAT_CULTURE 의 원래 요청금액)
4. **LIVE 분기**: 컬쳐랜드가 보낸 form 파라미터 (`RetVal`/`UserID`/`CustID`/`HashNo`/`McashAmount`/`RequestAmount`) 그대로 사용
5. `CultureMapper.updateTbatCulture` 로 TBAT_CULTURE 갱신
6. 결과 페이지 (`culture/cultureReturn.html`) 렌더

**상수**:
- `MOCK_MEMBER_CODE = "MOCKCULTURE1"` (Phase 1 v02 SQL 의 MOCK 가맹점코드와 일치)
- `MOCK_MCASH_AMT = 1_000_000` (테스트 잔액)

**로그**: `[cultureReturn] MOCK branch — nonce=..., mbrCd=..., reqAmt=...` / `[cultureReturn] REAL branch — nonce=..., mbrCd=..., retVal=...`
